Chapter02 MySQL 的調控按鈕 -- 啟動選項和系統變數
=====
* ### 預設最大用戶端同時連線數為 151 + 1 (超級使用者總是需要特權)。
* ### 預設儲存引擎為 InnoDB。
* ### 選項的長短形式
| 長形式 | 短形式 | 含義 |
| --- | --- | --- |
| --host | -h | 主機名稱 |
| --user | -u | 用戶名 |
| --password | -p | 密碼 |
| --port | -P | 通訊埠 |
| --version | -V | 版本資訊 |
* ### 命令列的設定只有該次有效。
* ### 在類 UNIX 系統中，每一個使用者都有一個使用者目錄，"~" 代表當前使用者的使用者目錄。
* ### 在設定檔中只能使用長形式選項，且不允許加上 "--" 前綴呦 !
* ### 設定檔中的選項群組
    * ### \[server\]: 作用於所有伺服器程式。
    * ### \[client\]: 作用於所有用戶端程式。
    * ### \[xxx\]: 作用於對應的程式。
* ### 設定檔中的選項群組可以加上特定的版本編號。
* ### 設定檔相互之間有優先順序的特性，設定檔內則是以最後出現的設定為主。
* ### 命令列的啟動選項優先度大於設定檔，也就是以命令列為主。
* ### 允許同時連入用戶端數量系統變數為 max_connections。
* ### 預設儲存引擎系統變數為 default_storage_engine。
* ### 查詢快取大小系統變數為 query_cache_size。
* ### 大部分的系統變數在運行過程中是可以更改的呦 !
* ### 部分系統變數可以獨立於用戶端，且使用時相互不影響。
* ### 系統變數作用域
    * ### GLOBAL (全域範圍): 影響伺服器整體。
    * ### SESSION (階段範圍): 影響某個用戶端。
    * ### 伺服器會為每一個用戶端維護獨立的 SESSION 作用域，在其一開始連線時使用 GLOBAL 的當前值進行該用戶端 SESSION 值的初始化，也就是複製一份出來給該用戶端專用的概念 (其實不是所有值都被複製)。
    * ### 透過啟動選項設定的系統變數都是 GLOBAL 的 (畢盡沒啟動伺服器就沒有用戶端嘛)。
    * ### 查看顯示時，如果查看 GLOBAL 作用域，其中未設定的變數不會被顯示; 但若查看 SESSION 作用域，未設定的變數會顯示 GLOBAL 作用域的值。
* ### 在運作過程中，若有某個用戶端修改 GLOBAL 的值，當前已連線的用戶並不會受到影響，只有後續連入的用戶其 SESSION 值會使用新的 GLOBAL 值。
* ### 注意 (只是舉例不只這些)
    * ### 只有 GLOBAL 作用域的系統變數: max_connextions, query_cache_size。
    * ### 只有 SESSION 作用域的系統變數: AUTO_INCREMENT。
    * ### GLOBAL 作用域與 SESSION 作用域都有的系統變數: default_storage_engine。
    * ### 還有些是唯讀的，無法設置: version。
* ### 啟動選項是使用者傳遞的參數，系統變數是影響伺服器運行的變數，大部分系統變數可以作為啟動選項，但有些系統變數是運行過程中產生的就不可以摟，當然也有些啟動選項不是系統變數。
* ### 狀態變數 (MySQL 伺服器在程式運行時維護的變數)
    * ### Threads_connexted: 當前連接用戶數。
    * ### Innodb_rows_updated: 更新了多少筆以 InnoDB 為儲存引擎的表中的紀錄。
    * ### 也有 GLOBAL 和 SESSION 的差別喔。
