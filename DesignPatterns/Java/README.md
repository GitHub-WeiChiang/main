設計守則
=====
* ### 找出程式中可能需要更動之處，把它們獨立出來，不要和那些不需要更動的程式碼混在一起。
* ### 寫程式是針對介面而寫，而不是針對實踐方式而寫。
* ### 多用合成，少用繼承。
* ### 設計時，盡量讓需要互動的物件之間關係鬆綁。
* ### 類別應該開放，以便擴充；應該關閉，禁止修改。
* ### 依賴抽象類別，不要依賴具象類別。
* ### 認識極少化守則，只和你的密友談話。
* ### 別呼叫（打電話給）我們，我們會呼叫（打電話給）你。
* ### 一個類別應該只具有一個改變的理由。
