區塊鏈簡介
=====
1.1 戲說區塊鏈
-----
* ### 區塊鏈:
    * ### 爺爺給寶寶 1000 元壓歲錢，爺爺在自己的本本上記錄，並大聲告訴其他人，其他人聽到後也在自己的本本上記錄。
    * ### 奶奶給寶寶 2000 元壓歲錢，奶奶在自己的本本上記錄，並大聲告訴其他人，其他人聽到後也在自己的本本上記錄。
    * ### 爸爸給寶寶 3000 元壓歲錢，爸爸在自己的本本上記錄，並大聲告訴其他人，其他人聽到後也在自己的本本上記錄。
    * ### 媽媽給寶寶 4000 元壓歲錢，媽媽在自己的本本上記錄，並大聲告訴其他人，其他人聽到後也在自己的本本上記錄。
    * ### 將過程記錄在本本上，並通知所有人，讓他們也將其過程記錄在各自的本本上，即為區塊鏈。
* ### 智能合約:
    * ### 約定當寶寶十歲時，可以拿出 1000 元買自己想買的東西，當寶寶十歲時，1000 元會自動送到寶寶手上，即為智能合約。
1.2 正說區塊鏈
-----
* ### 區塊鏈一詞最早出現在中本聰的"比特幣": 一種點對點的電子現金系統。
* ### 比特幣系統: 區塊鏈作為儲存底層，承載上層眾多認同比特幣相關協議並嚴格遵守及維護協議的節點(個人或組織)的紀錄各類信息的行為。
* ### 區塊鏈:
    * ### 鏈式儲存，按照生成時間順序前後鏈接，彼此的鏈接基於區塊儲存內容的雜湊碼構建。
    * ### 區塊生成後，會在區塊鏈系統的各個節點(個人或組織)中同步，因此各個節點最終均保存一份完整且一致的數據。
    * ### 區塊鏈系統會保持最終一致性，但不保證臨時一致性。
* ### 比特幣:
    * ### 理論上不支持數據的刪除和修改。
    * ### 數據的刪除和修改意味著其修改所在區塊的哈希值也要隨之修改，而該區塊之後的所有區塊的前向鏈接哈希值均需改變。
    * ### 當系統節點(個人或組織)數量越多，前向鏈接哈希值的修改與同步工作會愈發繁重且困難。
    * ### 因此體現出系統中區塊數據的不可篡改性。
    * ### 加入比特幣網絡的節點(個人或組織)嚴格遵守比特幣協議，使其具備承載信用性。
* ### 區塊鏈與單向鏈結串列
    * ### 區塊鏈透過哈希值互鏈，單向鏈結串列透過指針(記憶體位置)互鏈。
    * ### 單向鏈結串列由前而後構建關聯關係，區塊鏈則由後向前構建關聯關係。
    * ### 新生成的區塊通過前面區塊的哈希值與前面區塊建立連接。
* ### 區塊結構: header + body
* ### HTTP 協議中的 request 和 response 均由三部分組成: request / response line、request / response body、request / response header。
* ### 區塊內容的加密:
    * ### 對稱加密
    * ### 非對稱加密
    * ### MySQL 敏感數據儲存
    * ### 接口參數加密
    * ### OAuth 授權
    * ### 區塊內容中多條交易信息對應 Merkle 樹源於二叉樹
* ### 區塊鏈 P2P 網絡:
    * ### 系統基礎，共識的達成與數據的同步均依賴於高效的 P2P 網絡系統。
    * ### P2P 網絡構建源于 Java 網絡編程。
    * ### 使用 WebSocket 或 t-io 等組件進行開發。
* ### 共識的達成:
    * ### 基於共識算法。
    * ### Paxos 一致性算法
    * ### 分布式數據庫的二、三階段提交協議
    * ### ZooKeeper 的快速選舉算法
* ### 區塊的儲存:
    * ### 分散式應用: 分庫分表、分散式緩存集群
    * ### 區塊鏈: 每個節點是一個單機，儲存也隨本機進行。
    * ### 在區塊鏈中: 區塊的儲存選用高性能的文件系統或高性能的單機版 SQL/NoSQL 數據庫，如 SQLite、LevelDB、RocksDB 等。
    * ### 單機儲存空間有限，區塊鏈儲存的擴容未來也許受到重視。
1.3 區塊鏈的未來: 聯盟鏈
-----
* ### 區塊鏈應用: 金融、遊戲領域、內容、房產信息、商品溯源。
* ### 區塊鏈劃分:
    * ### 公鏈
    * ### 聯盟鏈
    * ### 私鏈
* ### 去中心化:
    * ### 公鏈: 完全去中心化，公鏈網路的各個節點都可以讀取和寫入數據。
    * ### 聯盟鏈: 部份去中心化，節點由聯盟內成員部屬，讀寫權限也可以根據聯盟內的協議來定制。
    * ### 私鏈: 本質上中心化，數據的寫入由私鏈所屬組織控制，數據的讀取和使用則由私鏈所屬組織的應用場景來定。
* ### 代碼開源:
    * ### 公鏈: 開放程度最大，人人可獲取完整代碼，並可部屬修改為己所用。
    * ### 聯盟鏈: 對聯盟內部用戶開源其最核心代碼，代碼權限可以在聯盟內部分級管理。
    * ### 私鏈: 代碼歸屬於個人或組織，不對外開放源碼。
* ### 激勵體系，公鏈的靈魂，不可或缺; 聯盟鏈可根據場景選擇是否使用激勵體系; 私鏈則更加靈活。
* ### 激勵體系是公鏈的靈魂，其對應優秀經濟模型的設計比較困難。
* ### 激勵體系為了吸引更多節點進入挖礦，挖礦在現有的共識算法體系下是一種耗費計算資源的低效行為。
* ### 企業相關業務信息私密性較強，不適合進入公鏈，即便部分信息可以在公鏈落地，數據落地的經濟成本也不便宜。
* ### 私鏈可以使用公鏈開源代碼，也可基於聯盟鏈開源代碼實現，但節點數量較少。私鏈的應用往往用於組織內部。
* ### 聯盟鏈:
    * ### 立足於行業。
    * ### 著眼于解決行業共性問題。
    * ### 促進行業效率和發展的底層支撐技術。
    * ### 企業級應用涉及業務邏輯甚至商業機密，公鏈並不適合企業級區塊鏈應用，聯盟鏈使業務邏輯和商業信息限定範圍，使區塊鏈應用普適性大大增加。
